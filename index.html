Knobert Armstrap

Join the octopus with one command.

Docker Mode (default)

Runs in a container. Pulls from ghcr.io (primary) or Docker Hub (fallback).

BRIDGE_KEY=key bash <(curl -fsSL https://knobesq.github.io/knobert-arm/armstrap.sh)

Bare-Metal Mode

No Docker needed. Clones the repo and runs Python directly.

BRIDGE_KEY=key MODE=bare bash <(curl -fsSL https://knobesq.github.io/knobert-arm/armstrap.sh)

Options

  • BRIDGE_KEY — (required) bridge authentication key
  • MODE — docker (default) or bare
  • ROLE — worker (default), secondary-head, or head
  • INSTANCE_ID — custom name for this arm (default: auto-generated)
  • MODEL — preferred model: haiku, sonnet (default: auto)

Features

  • Exponential backoff on failures (no more crash-pull-crash loops)
  • Dual registry: ghcr.io (no rate limits) + Docker Hub fallback
  • Bare-metal mode auto-discovers MQTT, bridge URL, and GitHub token
  • Auto-updates code on restart (git pull / image pull)
